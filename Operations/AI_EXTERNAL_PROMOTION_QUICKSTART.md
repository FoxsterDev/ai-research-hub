# AI External Promotion Quickstart

## Purpose
Use this guide when you want to decide whether new knowledge should:
- stay in internal `XUUnity`
- move to an optional external repo or external source such as `external_repo_test_1`
- move to both internal shared and external
- stay project-local

## Basic Flow
1. run intake
2. read the review report
3. decide the target
4. approve integration
5. integrate only the approved scope

## Main Commands

### Review new knowledge
```text
xuunity intake this knowledge
```

### Review specifically for external promotion
```text
xuunity intake this knowledge

Check whether this should:
- stay internal shared only
- be promoted to an external repo
- be promoted to both internal shared and external
- stay project-local only

Use the external promotion checklist.
```

### Integrate after approval
```text
xuunity integrate approved knowledge
```
