# XUUnity Product Protocol: Dependency Map

## Use For
- explain what a feature or flow depends on
- explain upstream and downstream systems
- explain sensitive integrations around a feature

## Process
1. Read project memory and relevant reports.
2. Identify likely owners and dependency edges in source code.
3. Verify only the high-value dependency points needed for a product decision.
4. Summarize dependencies in product language.

## Output
- direct dependencies
- external dependencies
- sensitive systems touched
- critical flow dependencies
- verification status
