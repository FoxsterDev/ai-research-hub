# XUUnity Product Protocol: Bug Impact Brief

## Use For
- explain the product impact of a bug
- explain which users and flows are affected
- explain urgency from a business and rollout perspective

## Process
1. Read project memory and relevant prior reports.
2. Verify the affected flow in source code when possible.
3. Translate technical failure into user-visible and business-visible impact.
4. Keep the answer focused on severity, blast radius, and urgency.

## Output
- affected audience
- affected flow
- severity from product perspective
- likely business or retention impact
- rollout or hotfix urgency
- verification status
