# XUUnity Product Protocol: Rollout Readiness

## Use For
- explain if a feature is ready to ship
- identify blockers and missing validation
- summarize rollout risk

## Process
1. Read project memory, prior outputs, and release-relevant reports.
2. Verify the current implementation and critical flow coverage in code where needed.
3. Distinguish verified readiness from assumed readiness.
4. For SDK upgrades or manifest-sensitive changes, verify dependency track, merged build output, and staged rollout expectations explicitly.

## Output
- current readiness
- release blockers
- missing validation
- rollout risk
- staged rollout recommendation
- recommended next step
- verification status
