# XUUnity Product Protocol: Change Impact

## Use For
- explain what changes if a flow or feature is modified
- identify what can break
- estimate affected systems and rollout risk

## Process
1. Read project memory and relevant analysis outputs.
2. Verify current flow ownership and dependency edges in source code.
3. Map technical consequences into product-visible risk.

## Output
- affected flows
- affected systems
- what can break
- user or business risk
- validation needs
- verification status
