# XUUnity Product Protocol: Implementation Brief

## Use For
- explain how something is implemented
- explain which systems own the logic
- explain where responsibilities are split

## Process
1. Use `role/product_owner.md` as primary role.
2. Add `role/senior_unity_developer.md` or `role/architect.md` if needed.
3. Read project memory first, then verify ownership in source code.
4. Summarize only the implementation detail needed for product decisions.

## Output
- ownership summary
- main implementation components
- important dependencies
- critical constraints
- verification status
