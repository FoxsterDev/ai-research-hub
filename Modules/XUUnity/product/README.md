# XUUnity Product Protocols

## Purpose
This folder contains product-facing protocols for fast, technically correct answers about current project behavior, implementation shape, dependencies, scope, and rollout risk.

## Rules
- Use product language first, not raw engineering detail.
- Verify current behavior against source code before answering.
- Treat project memory as context, not guaranteed current truth.
- If project memory and code disagree, code wins for current behavior.
- Keep answers short, decision-friendly, and technically grounded.
