# XUUnity Utility: Internet Research Watch

## Goal
Periodically research what is new outside the repo and decide whether any of it should improve the current AI system, skills, workflows, or tooling.

## Use For
- watching for new Unity mobile best practices
- watching for new SDK integration patterns
- watching for new AI coding and review workflows
- watching for new tooling that can improve the current system
- checking whether current internal guidance is falling behind

## Research Focus
- Unity `6000+` mobile engineering practices
- Android and iOS integration and compliance changes
- AI-assisted engineering workflows
- coding agent and IDE tooling improvements
- review automation patterns
- test automation patterns
- product-owner self-service patterns

## Process
1. Define the watch topic and time horizon.
2. Gather new external signals from primary sources when possible.
3. Classify findings into:
   - direct improvement to current `XUUnity`
   - possible experiment
   - optional external repo candidate such as `external_repo_test_1`
   - no action
4. Compare each finding against current internal tools and prompts.
5. Recommend:
   - adopt now
   - test in one project
   - keep under watch
   - reject

## Output
- watch topic
- key new findings
- why they matter now or why they do not
- impacted internal tools or prompt areas
- recommendation
- whether to run `knowledge_intake_review.md` next

## Rule
Do not chase novelty for its own sake.
Only recommend changes that improve safety, speed, consistency, verification quality, or portfolio scalability.
