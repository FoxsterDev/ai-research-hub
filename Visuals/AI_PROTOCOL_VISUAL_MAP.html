AI Protocol Visual Map

This page summarizes the public AIRoot routing model. It documents the public xuunity core, generic optional host-local overlays, and project-local memory boundaries.

1. Protocol Families

Repo Router (Agents.md)
  -> xuunity
  -> optional host-local protocols

The public module names only xuunity. Host-local families may exist, but public AIRoot refers to them only generically.

2. Load Order

Repo Agents.md
  -> Public XUUnity Core (AIRoot/Modules/XUUnity/)
    -> Optional Host-Local Overlays (AIModules/ when attached)
      -> Project Agents.md
        -> ProjectMemory (Assets/AIOutput/ProjectMemory/)
          -> Previous Outputs (Assets/AIOutput/)

3. Knowledge Flow

New Knowledge
  -> Extract
    -> Intake Review
      -> public-safe reusable      -> AIRoot/Modules/XUUnity/
      -> host-local reusable       -> AIModules/
      -> project-specific          -> ProjectMemory / SkillOverrides
      -> report or draft           -> Assets/AIOutput/
      -> reject                    -> No Action

4. Storage Boundaries

Layer Purpose
AIRoot Public reusable modules, setup guidance, templates, and visuals.
AIModules Optional host-local overlays and private protocol families.
AIOutput Host-local mutable registry, reports, wrappers, and operational notes.
<Project>/Assets/AIOutput/ProjectMemory/ Project-local durable truth and overrides.
<Project>/Assets/AIOutput/ Project-local reports, audits, and drafts.

5. Quick Routing Guide

Need Route
Fix or refactor code xuunity ...
Review SDK or native integration xuunity sdk ... or xuunity native ...
Ask product-facing implementation question xuunity product ...
Audit project readiness xuunity product health ...
Check project-memory freshness xuunity project memory freshness ...
Use private host-local flows Host-specific protocol selected by the host router
